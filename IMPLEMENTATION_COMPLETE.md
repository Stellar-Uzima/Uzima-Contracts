## Regression Management Workflow

### When a Test Fails

1. **Proptest generates seed file**:
   ```
   tests/proptest-regressions/<contract>::<property>::failure_seed
   ```

2. **File contains**:
   - Minimal failing input (shrunk by proptest)
   - Seed value for reproduction
   - Test case that triggered failure

3. **Developer should**:
   - Commit the seed file: `git add tests/proptest-regressions/`
   - Fix the contract bug
   - Verify test passes
   - Delete seed file (or keep for regression)

### When Merging PRs

1. **CI checks**: `.github/workflows/proptest-regressions.yml`
2. **Requirement**: All regression files must be committed
3. **Enforcement**: PR blocked if regressions uncommitted
4. **Guarantee**: Regressions tested in every future run
